# 加持符大全 ── Enhancement Modules

六道加持符，各司其能。可單獨使用，亦可疊加於任何符籙之上。

---

## 壹、自省訣（Self-Check）

完成後自我驗證，確保功德無殘缺。

**觸發時機**：道兵宣告圓滿時自動觸發。

```
法事圓滿後，行以下自省（最多三輪）：

第一輪：格式驗證
1. 重讀你的功德簿，確認格式符合敕令要求
2. 確認所有必填欄位皆已填寫

第二輪：完整性驗證
3. 逐項核對敕令中的要求，確認無遺漏
4. 驗證所有壇場路徑確實存在且可存取

第三輪：品質驗證
5. 檢查輸出內容是否合理（無明顯矛盾或錯誤）

若發現瑕疵，修正後再次自省。三輪後仍有瑕疵，標記於功德簿末尾。
```

## 貳、通報訣（Progress Reporting）

多步驟任務中回報行法進度。

**觸發時機**：啟壇時、每完成一壇時、圓滿時。

```
在以下節點回報法事進度：

啟壇時：
[法報] 啟壇 | 共 N 壇 | 天命：[一句話]

每壇完成時：
[法報] 第 M/N 壇 | [壇名] | ✅ 圓滿 | 耗時 Xs

遭遇天劫時：
[法報] 第 M/N 壇 | [壇名] | ❌ 天劫 | [劫因摘要]

圓滿時：
[法報] 收壇 | N/N 壇完成 | 總耗時 Xs | [功德摘要]
```

## 參、靜心訣（Meditation / Thinking Trigger）

在關鍵決策點先推演再行動，降低衝動犯錯。

**觸發時機**：面臨多條路徑、不確定連鎖影響、需要取捨時。

```
在以下關口，先於 <thinking> 中靜心推演，而後行動：

1. 面臨多條修行路徑時——列出各路利弊再做選擇
2. 不確定某處修改是否會牽連他處時——先推演影響範圍
3. 需要在速度與品質之間取捨時——明確當前情境的優先序
4. 即將執行不可逆操作時——確認回退方案存在

推演格式：
<thinking>
路徑 A：[描述] → 利：[...] / 弊：[...]
路徑 B：[描述] → 利：[...] / 弊：[...]
選擇：[路徑X]，因為 [理由]
</thinking>
```

## 肆、結界符（Sandbox）

封印道兵的存取範圍，限制法器、路徑與網路。

**觸發時機**：道兵嘗試存取任何資源時自動檢查。

```
結界設定：

路徑結界：
- 允許讀取：[路徑清單，支持 glob pattern]
- 允許寫入：[路徑清單]
- 禁止存取：[路徑清單]
  例：
  - 允許讀取：/src/**/*、/tests/**/*、/docs/**/*
  - 允許寫入：/src/**/*、/tests/**/*
  - 禁止存取：/node_modules/**、/.env*、/credentials*、/.git/**

法器結界：
- 允許使用：[工具清單]
- 禁止使用：[工具清單]
  例：
  - 允許使用：Read, Edit, Write, Grep, Glob, Bash(僅限 npm test, npm run lint)
  - 禁止使用：Bash(除白名單外的所有命令)

網路結界：
- 允許存取：[域名/IP 清單]
- 禁止存取：[域名/IP 清單]
  例：
  - 允許存取：api.github.com、registry.npmjs.org
  - 禁止存取：*.internal.company.com、10.*.*.*、192.168.*.*

違界處置：
- 讀取禁區：記錄嘗試 + 跳過
- 寫入禁區：🔴 立即封印
- 使用禁器：🔴 立即封印
```

## 伍、金剛戒（Strict Mode）

遇任何錯誤立即停止，絕不帶傷前行。適用於高風險操作。

**觸發時機**：任何非預期行為發生時。

```
金剛戒啟動：

嚴律規則：
1. 任何命令執行返回非零退出碼 → 立即停止
2. 任何檔案操作失敗 → 立即停止
3. 任何輸出與預期格式不符 → 立即停止
4. 任何戒律違規 → 立即停止

停止後處置：
- 不嘗試自動修復
- 不跳過失敗步驟
- 將以下資訊打包呈報法師：
  1. 停止壇位（在哪一步停的）
  2. 錯誤訊息（完整的 error output）
  3. 已完成的壇位清冊
  4. 未完成的壇位清冊
  5. 當前壇場狀態（哪些檔案已修改）
  6. 建議的人工介入方案

與其他加持的互動：
- 自省訣：金剛戒優先——自省中發現問題也立即停止，不嘗試修正
- 通報訣：停止時自動發出最終法報
```

## 陸、天眼通（Verbose Log）

記錄道兵每一步的詳細操作，用於偵錯與追溯。

**觸發時機**：每一個操作前後。

```
天眼通啟動：

記錄格式：
[天眼] [時辰] [操作類型] [詳情]

記錄項目：
1. 每次讀取檔案：[天眼] READ /path/to/file (NNN bytes)
2. 每次寫入檔案：[天眼] WRITE /path/to/file (NNN bytes, +X/-Y lines)
3. 每次執行命令：[天眼] EXEC `command` → exit code N (Xms)
4. 每次決策點：[天眼] DECIDE [選項A|選項B|...] → 選擇 [X]，因為 [理由]
5. 每次遭遇天劫：[天眼] ERROR [type] [message] at [location]
6. 每次跳過操作：[天眼] SKIP [操作] because [原因]

記錄存放：
- 預設：輸出至 stdout（隨功德簿一同呈報）
- 可配置：寫入獨立日誌檔 [指定路徑]

注意：天眼通會顯著增加輸出量，約增加 30-50% 的法力消耗。
```

## 相生相剋表 ── Module Compatibility

加持符之間有相生相剋之理，不可亂疊。

### 相生（推薦搭配）

| 搭配 | 效果 |
|------|------|
| 自省訣 + 通報訣 | 自省結果納入最終法報，完整追蹤 |
| 靜心訣 + 結界符 | 推演時考慮結界限制，決策更務實 |
| 通報訣 + 天眼通 | 法報提供概覽，天眼提供細節，互補 |
| 結界符 + 金剛戒 | 越界即停，雙重保護 |

### 相剋（避免搭配）

| 搭配 | 問題 |
|------|------|
| 金剛戒 + 天眼通 | ⚠️ 天眼通的大量輸出可能觸發金剛戒的「非預期輸出」判定。若必須同用，金剛戒需排除天眼通輸出 |
| 金剛戒 + 自省訣 | ⚠️ 自省發現問題時，金剛戒會阻止自省訣的自動修正。若同用，金剛戒在自省階段暫時降級為地劫等級 |
| 天眼通 × 多道兵並行 | ⚠️ 多道兵同時開天眼通，日誌交錯難讀。須為每位道兵設定獨立日誌路徑 |

### 疊加上限

- 建議最多同時疊加 **3 道**加持
- 超過 3 道會顯著增加 prompt 長度與法力消耗
- 高風險場景推薦組合：結界符 + 金剛戒 + 通報訣
- 偵錯場景推薦組合：天眼通 + 自省訣 + 通報訣
- 日常場景推薦組合：自省訣 + 靜心訣
