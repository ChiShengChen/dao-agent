# 法力估算 ── Token Budget Estimation

書符之前，先估法力。符籙太短如殘符無力，太長如亂咒反噬。此法門幫助估算合理的 token 預算。

---

## 法力計量

### 符籙本身的法力（Prompt Tokens）

| 符籙類型 | 建議長度 | 約略 tokens |
|---------|---------|------------|
| 飛符（一次性 `claude -p`） | 簡短精煉 | 150–400 |
| 獨行道兵符 | 適中 | 300–600 |
| 千里眼 / 造化真人 / 監察御史 | 中等偏詳 | 400–800 |
| 都天大法主符 | 詳盡 | 600–1200 |
| 鎮壇符（CLAUDE.md） | 完整 | 500–1500 |
| 大醮科儀（多符組合） | 各符分開計 | 每符 300–800 |

### 道兵執行的法力（Completion Tokens）

根據任務複雜度估算道兵需要的回應空間：

| 任務類型 | 預估回合數 | 每回合 tokens | 總法力 |
|---------|-----------|-------------|-------|
| 簡單查詢/搜尋 | 1–3 | 500–1000 | 1K–3K |
| 單檔修改 | 2–5 | 1000–2000 | 2K–10K |
| 多檔重構 | 5–15 | 1500–3000 | 8K–45K |
| 完整功能開發 | 10–25 | 2000–4000 | 20K–100K |
| 大型科儀（多 agent） | 各 agent 分計 | — | 50K–500K |

---

## 估算公式

### 單一道兵

```
總法力 ≈ 符籙法力 + (回合數 × 每回合法力) + 法器呼叫開銷

其中：
- 符籙法力：依上表估算
- 回合數：依任務複雜度
- 每回合法力：依產出物大小
- 法器呼叫開銷：每次工具呼叫約 200–500 tokens（含輸入輸出）
```

### 九轉金丹（迭代）

```
總法力 ≈ 轉數 × (造化真人法力 + 監察御史法力)

通常：
- 一轉成丹：2x 單次法力
- 二轉成丹：4x 單次法力
- 三轉成丹：6x 單次法力

建議預算按最大轉數的 80% 估算（多數不會真的煉到最後一轉）
```

### 五雷正法（並行）

```
總法力 ≈ 先鋒法力 + Σ(各路雷神法力) + 匯雷法力

注意：並行不省 tokens，只省時間。
N 路並行的總法力 ≈ N × 單路法力
```

### 天地人三才（分層）

```
總法力 ≈ 天層(決策) + Σ(人層各將) + 地層(監控)

天層通常最輕（只做分派）：~2K
人層為主要消耗：各將分別估算
地層視監控頻率而定：~1K–5K
```

## 快速估算表

回答以下問題，查表得出預算：

### 第一問：幾位神將？

| 神將數 | 基礎倍率 |
|--------|---------|
| 1 位 | ×1 |
| 2–3 位 | ×2.5 |
| 4–6 位 | ×5 |
| 7+ 位 | ×8+ |

### 第二問：任務有多複雜？

| 複雜度 | 基礎法力 | 典型場景 |
|--------|---------|---------|
| 簡（一步完成） | 2K | 搜尋、簡單修改、格式轉換 |
| 中（三五步驟） | 10K | 多檔修改、生成測試、程式碼審查 |
| 繁（十步以上） | 30K | 完整功能開發、大規模重構 |
| 極（開放探索） | 80K+ | 架構設計、複雜 debug、全專案分析 |

### 第三問：有迭代嗎？

| 迭代模式 | 追加倍率 |
|---------|---------|
| 無迭代 | ×1 |
| 最多 2 轉 | ×1.8 |
| 最多 3 轉 | ×2.5 |
| 最多 5 轉 | ×4 |

### 計算

```
預估總法力 = 基礎法力 × 神將倍率 × 迭代倍率

範例：
- 3 位神將做中等任務，無迭代：10K × 2.5 × 1 = 25K tokens
- 1 位道兵做繁重任務，3 轉迭代：30K × 1 × 2.5 = 75K tokens
- 5 位神將做複雜任務，2 轉迭代：30K × 5 × 1.8 = 270K tokens
```

## 法力節省之術

### 壹、選對模型

| 場景 | 建議模型 | 法力單價 |
|------|---------|---------|
| 簡單搜尋、格式轉換 | claude-haiku-4-5 | 低 |
| 一般開發、審查 | claude-sonnet-4-6 | 中 |
| 複雜推理、架構決策 | claude-opus-4-6 | 高 |

在符籙中可指定：
```
法力配置：
- 千里眼（探查）：使用 haiku，省法力
- 造化真人（核心開發）：使用 sonnet
- 都天大法主（決策）：使用 opus
```

### 貳、精簡符文

- 刪除冗餘的解釋和背景，只留敕令
- 用簡潔的格式定義取代冗長描述
- 範例只給一組，不要三組

### 參、縮小壇場

- 明確指定目標檔案，不要讓道兵全壇場搜尋
- 用 Glob pattern 限縮範圍：`src/**/*.ts` 而非 `**/*`
- 先讓千里眼探查，再讓造化真人針對性動手

### 肆、善用 max_turns

- 設定合理的 `max_turns` 防止道兵無限行法
- 簡單任務：3–5 turns
- 中等任務：5–15 turns
- 複雜任務：15–30 turns
