# 符籙驗證 ── Prompt Linting Rules

符籙書成之後，須經此法門驗證。殘符無力，亂咒反噬——驗證確保每道符籙結構完備、指令明確。

---

## 驗符科儀

生成符籙後，自動按以下檢查項逐一過堂。分為三級：

### 🔴 天劫級（必修——缺此不可出壇）

| # | 檢查項 | 判定法則 | 化解之方 |
|---|--------|---------|---------|
| 1 | **天命明確** | 符籙中是否有一句話明確說明目標？能否回答「這位道兵要做什麼？」 | 在符頭加入「天命：[一句話]」 |
| 2 | **功德簿有定** | 是否定義了輸出格式與存放位置？ | 加入「功德簿」區段，指明格式與路徑 |
| 3 | **圓滿條件** | 是否有明確的完成判定？道兵怎麼知道自己做完了？ | 加入「圓滿條件：當 [X] 且 [Y] 時」 |
| 4 | **禁制存在** | 是否至少有一條禁制？無禁制的道兵如脫韁野馬 | 加入「禁制」區段，至少寫明不可做的事 |
| 5 | **無含糊指令** | 是否存在「適當地」「相關的」「合理的」等含糊用語？ | 替換為具體指令：「適當地處理錯誤」→「遇錯誤時記入 /tmp/err.log 並繼續」 |

### 🟡 地劫級（宜修——缺此恐有隱患）

| # | 檢查項 | 判定法則 | 化解之方 |
|---|--------|---------|---------|
| 6 | **渡劫之策** | 是否定義了錯誤處理？至少涵蓋最可能的失敗情境 | 加入「渡劫之策」區段 |
| 7 | **法器清單** | 是否列出可用與禁用的工具？ | 加入「法器清單」，分「可持用」與「不可觸碰」 |
| 8 | **科儀步驟** | 是否有 numbered steps 或 decision tree？ | 將散文式指令改為編號步驟 |
| 9 | **因緣交代** | 是否提供了足夠的上下文？道兵是否理解「為何」要做此事？ | 加入「因緣」區段，一到兩句話 |
| 10 | **範例附錄** | 對於格式要求嚴格的功德簿，是否附有 input/output 範例？ | 附上至少一組範例 |

### 🔵 人劫級（可修——修之更佳）

| # | 檢查項 | 判定法則 | 化解之方 |
|---|--------|---------|---------|
| 11 | **符文長度** | 是否在 300–800 tokens 之間？ | 過短則補充細節，過長則精簡或拆分 |
| 12 | **祈使語氣** | 是否使用敕令口吻（祈使句）？ | 「你應該讀取檔案」→「讀取檔案」 |
| 13 | **自省訣** | 是否包含完成後的自我驗證步驟？ | 在末尾加入自省訣 |
| 14 | **靜心訣** | 關鍵決策點是否有 thinking 提示？ | 在複雜判斷前加入靜心訣 |
| 15 | **通報訣** | 多步驟任務是否有進度回報機制？ | 加入通報訣 |

## 驗符奏摺格式

驗證完成後，輸出以下格式的報告：

```markdown
# 符籙驗證奏摺

## 符籙概要
- 法號：[角色名]
- 天命：[一句話目標]
- 符文長度：約 [N] tokens

## 驗證結果

| 等級 | 通過 | 未通過 | 總計 |
|------|------|--------|------|
| 🔴 天劫 | N | N | 5 |
| 🟡 地劫 | N | N | 5 |
| 🔵 人劫 | N | N | 5 |

## 未通過項目

### 🔴 天劫
- [ ] #N [檢查項名]：[具體問題] → [修復建議]

### 🟡 地劫
- [ ] #N [檢查項名]：[具體問題] → [修復建議]

### 🔵 人劫
- [ ] #N [檢查項名]：[具體問題] → [修復建議]

## 判定
- ✅ 可出壇（天劫全過）
- ⚠️ 帶傷出壇（天劫全過，地劫有缺）
- ❌ 不可出壇（有天劫未過）
```

## 自動驗符觸發

在生成符籙的流程中，於「第三步：書符敕令」完成後自動執行驗符。流程如下：

```
書符完成 → 自動驗符 → 天劫全過？
                        ├─ 是 → 呈報奏摺 + 輸出符籙
                        └─ 否 → 自動修補 → 再次驗符（最多三輪）
                                              └─ 仍未過 → 呈報奏摺 + 標註未過項 + 輸出符籙
```

## 常見殘符範例

**殘符（驗證不過）：**
```
你是一個助手。幫我處理一下那些檔案，輸出結果。
```

**問題：**
- 🔴 #1 天命不明：「處理一下」是什麼意思？
- 🔴 #2 功德簿未定：「輸出結果」是什麼格式？放哪裡？
- 🔴 #3 圓滿條件缺失：怎麼算做完？
- 🔴 #4 禁制缺失：沒有任何限制
- 🔴 #5 含糊指令：「那些檔案」是哪些？

**修正後：**
```
你是一位資料淨化真人。

天命：清洗 /data/raw/ 下所有 CSV 檔案，移除重複行與空值列。

科儀：
1. 列出 /data/raw/*.csv 所有檔案
2. 對每個檔案：移除完全重複的行、移除所有欄位皆為空的列
3. 清洗後的檔案存入 /data/clean/，保持原檔名

禁制：
- 不得修改 /data/raw/ 中的原始檔案
- 不得改變欄位順序或名稱
- 遇到編碼錯誤的檔案，跳過並記錄於 /data/clean/errors.log

功德簿：
- 清洗後的 CSV 存放於 /data/clean/
- 圓滿後輸出摘要：處理幾份、移除幾行、跳過幾份

圓滿條件：
所有 CSV 皆已處理（或記錄為錯誤），/data/clean/ 中檔案數 = 原始數 - 錯誤數。
```
